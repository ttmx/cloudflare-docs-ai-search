You can customize how your AI Search instance indexes your data, retrieves results, and generates responses. Some settings can be updated after the instance is created, while others are fixed at creation time.

## Data source

| Configuration                                                              | Editable after creation | Description                                              |
| -------------------------------------------------------------------------- | ----------------------- | -------------------------------------------------------- |
| [Built-in storage](/ai-search/configuration/data-source/built-in-storage/) | n/a                     | Upload files directly to an instance                     |
| [Website](/ai-search/configuration/data-source/website/)                   | no                      | Connect a domain you own to index website pages          |
| [R2 Bucket](/ai-search/configuration/data-source/r2/)                      | no                      | Connect a Cloudflare R2 bucket to index stored documents |

## Indexing

| Configuration                                                             | Editable after creation | Description                                                     |
| ------------------------------------------------------------------------- | ----------------------- | --------------------------------------------------------------- |
| [Vector search](/ai-search/configuration/indexing/vector-search/)         | yes                     | Vector search and the built-in vector index                     |
| [Path filtering](/ai-search/configuration/indexing/path-filtering/)       | yes                     | Include or exclude specific paths from indexing                 |
| [Chunking](/ai-search/configuration/indexing/chunking/)                   | yes                     | Number of tokens per chunk and overlap between chunks           |
| [Syncing](/ai-search/configuration/indexing/syncing/)                     | yes                     | Sync jobs and indexing controls                                 |
| [Keyword search](/ai-search/configuration/indexing/keyword-search/)       | yes                     | Enable keyword (BM25) search for exact term matching            |
| [Hybrid search](/ai-search/configuration/indexing/hybrid-search/)         | yes                     | Combine vector and keyword search with configurable fusion      |
| [Metadata attributes](/ai-search/configuration/indexing/metadata/)        | yes                     | Define built-in and custom metadata fields                      |
| [Service API token](/ai-search/configuration/indexing/service-api-token/) | yes                     | API token that grants AI Search permission to access R2 buckets |

## Retrieval

| Configuration                                                            | Editable after creation | Description                                                   |
| ------------------------------------------------------------------------ | ----------------------- | ------------------------------------------------------------- |
| [Result controls](/ai-search/configuration/retrieval/result-controls/)   | yes                     | Match threshold and maximum number of results                 |
| [Filtering](/ai-search/configuration/retrieval/filtering/)               | yes                     | Filter results by metadata attributes                         |
| [Relevance boosting](/ai-search/configuration/retrieval/boosting/)       | yes                     | Bias results by metadata characteristics                      |
| [Reranking](/ai-search/configuration/retrieval/reranking/)               | yes                     | Reorder results by semantic relevance using a reranking model |
| [Query rewriting](/ai-search/configuration/retrieval/query-rewriting/)   | yes                     | Rewrite follow-up queries using conversation context          |
| [System prompt](/ai-search/configuration/retrieval/system-prompt/)       | yes                     | Guide query rewriting and response generation behavior        |
| [Similarity caching](/ai-search/configuration/retrieval/cache/)          | yes                     | Cache responses for similar prompts                           |
| [Public endpoint](/ai-search/configuration/retrieval/public-endpoint/)   | yes                     | Enable public access to search, chat, and MCP endpoints       |
| [UI snippets](/ai-search/configuration/retrieval/embed-search-snippets/) | yes                     | Embed pre-built search and chat components in your website    |

## Models

| Configuration                                             | Editable after creation | Description                                         |
| --------------------------------------------------------- | ----------------------- | --------------------------------------------------- |
| [Embedding model](/ai-search/configuration/models/)       | no                      | Model used to generate vector embeddings            |
| [Generation model](/ai-search/configuration/models/)      | yes                     | Model used to generate the final response           |
| [Query rewriting model](/ai-search/configuration/models/) | yes                     | Model used for query rewriting                      |
| [Reranking model](/ai-search/configuration/models/)       | yes                     | Model used to reorder results by semantic relevance |
