[Private network routes](/cloudflare-one/networks/connectors/cloudflare-tunnel/private-net/cloudflared/) allow users to connect to your {props.networkName} using the Cloudflare One Client. To add a private network route for your Cloudflare Tunnel:

1.  In the Cloudflare dashboard, go to **Networking** > **Routes**.

    <DashButton url="/?to=/:account/magic-networks/routes" />

2.  Select **Create route** > **Tunnel CIDR**. Select the tunnel you just created, enter the **Private IP address** of your {props.resourceName} (for example, <code>{props.exampleIp}</code>), and select **Create route**. You can expand the IP range later if necessary.
3.  In your [Split Tunnel configuration](/cloudflare-one/team-and-resources/devices/cloudflare-one-client/configure/route-traffic/split-tunnels/#add-a-route), make sure the private IP is routing through the Cloudflare One Client. For example, if you are using Split Tunnels in **Exclude** mode, delete <code>{props.splitTunnelCidr}</code>. We recommend re-adding the IPs that are not explicitly used by your {props.resourceName}.

        To determine which IP addresses to re-add, subtract your {props.resourceName} IPs from <code>{props.splitTunnelCidr}</code>:

        <SubtractIPCalculator
        	client:load
        	defaults={{
        		base: props.splitTunnelCidr,
        		subtract: props.calcExclude
        	}}
        />

        Add the results back to your Split Tunnel Exclude mode list.

4.  To test on a user device:
    1. [Log in to the Cloudflare One Client](/cloudflare-one/team-and-resources/devices/cloudflare-one-client/deployment/manual-deployment/).
    2. Open a terminal window and connect to the service using its private IP:

    <Code code={`curl ${props.exampleIp}`} lang="sh" />

    ```txt output
    <html><body><h1>Hello Cloudflare!</h1>
    <p>This page was created for a Cloudflare demo.</p>
    </body></html>
    ```

You can optionally [create Gateway network policies](/cloudflare-one/networks/connectors/cloudflare-tunnel/private-net/cloudflared/#4-recommended-filter-network-traffic-with-gateway) to control who can access the {props.resourceName} via its private IP.
