DHCP Relay provides a way for DHCP clients to communicate with DHCP servers that are not available on the same local subnet/broadcast domain. When you enable DHCP Relay, Cloudflare One Appliance (formerly Magic WAN Connector) forwards DHCP discover messages to a predefined DHCP server, and routes the responses back to the original device that sent the discover message.

:::caution
DHCP relay will not work if your DHCP server is behind a <a href={props.cfTunnelURL}>Cloudflare Tunnel</a>. To enable DHCP relay functionality, use either a {props.cfVsCoreDashNamingMwanCni}.
:::

Create a [`PUT` request](/api/resources/magic_transit/subresources/sites/subresources/lans/methods/update/) to update the LAN where you want to enable DHCP relay:

Example:

<APIRequest
  path="/accounts/{account_id}/magic/sites/{site_id}/lans/{lan_id}"
  method="PUT"
  json={{
    "lan": {
    "static_addressing": {
      "dhcp_relay": {
        "server_addresses": [
          "192.0.2.1"
