[Browser Rendering](/browser-run/) REST API rate limits for Workers Paid plans have been increased from 3 requests per second (180/min) to **10 requests per second (600/min)**. No action is needed to benefit from the higher limit.

![Browser Rendering REST API rate limit increased from 3 to 10 requests per second](~/assets/images/changelog/browser-run/rest-api-limit-increase.png)

The [REST API](/browser-run/quick-actions/) lets you perform common browser tasks with a single API call, and you can now do it at a higher rate.

- [/content - Fetch HTML](/browser-run/quick-actions/content-endpoint/)
- [/screenshot - Capture screenshot](/browser-run/quick-actions/screenshot-endpoint/)
- [/pdf - Render PDF](/browser-run/quick-actions/pdf-endpoint/)
- [/markdown - Extract Markdown from a webpage](/browser-run/quick-actions/markdown-endpoint/)
- [/snapshot - Take a webpage snapshot](/browser-run/quick-actions/snapshot/)
- [/scrape - Scrape HTML elements](/browser-run/quick-actions/scrape-endpoint/)
- [/json - Capture structured data using AI](/browser-run/quick-actions/json-endpoint/)
- [/links - Retrieve links from a webpage](/browser-run/quick-actions/links-endpoint/)

If you use the [Browser Sessions](/browser-run/#integration-methods) method, increases to concurrent browser and new browser limits are coming soon. Stay tuned.

For full details, refer to the [Browser Rendering limits page](/browser-run/limits/).
